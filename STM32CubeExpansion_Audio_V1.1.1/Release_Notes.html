Release Notes of X-Cube-Audio Firmware Package

for STM32Cube Audio library software expansion

Copyright 2019 STMicroelectronics

 


STM32Cube is an STMicroelectronics original initiative to ease developers life by reducing development efforts, time and cost. STM32Cube covers STM32 portfolio.

 

STM32Cube Audio Expansion is a set of software libraries allowing customers to build audio applications with high-quality and professional results. The package offers highly optimized Audio Effects required in embedded audio solutions to deliver a high-quality listening experience, such as Bass Management, Biquad Filtering with application examples e.g. Loudness Control, clock drift compensation, Gain management, audio Virtualization for headphones and loudspeakers output, Graphical Equalization, Sampling Rate Conversion, Sound detector, Sound meter and Smart Volume Control solution. The package also embeds MP3 codec libraries to encode or decode an Audio stream in MP3 format (WAV format by default).


Major features of the STM32 Audio Firmware package

  • Audio Processing Effects and dynamic parameters interface
  • Sampling Rate Conversion from 8/16/24/32/96 kHz to 48 kHz, for more details refer to UM1641
  • Sampling Rate Conversion from 44.1 kHz to 48 kHz, for more details refer to UM1640
  • CDC - Clock Drift compensation for more details refer to UM1995
      • cdc mode: to add or remove one sample depending on drift direction
  • Biquad filtering with application usage examples and Biquad filter design tool, for more details refer to UM1625
      • enable: enable/disable audio processing
      • nb_sos: Number of second order sections
      • biquad_coeff[]: array of SOS coefficients and post-shift values.
      • Predefined filters available:
        • Notch: remove notch frequency at 1000Hz
        • Loudness: compensation of perceived loudness level versus frequency
        • Low pass: woofer low pass filter to remove frequencies above 150 Hz
        • High pass: high pass filter to remove frequencies below 150 Hz
        • Speech Enhancement: Improves dialogues perception
  • OmniSurround - Multichannel virtualization and Stereo Widening for loudspeakers, for more details refer to UM1633 and UM1655 respectively
      • Omni2Enable: Virtualization effect enable/disable
      • Omni2Strength: modifies the virtual position of the front speakers, opens the angle when strength increases (100% maximum widening position)
      • Omni2ListeningAngle: used to adapt listening angle from speakers distance
  • HPV - Headphone virtualization, for more details refer to UM2024
      • HpvEnable: virtualization effect enable/disable
      • HpvStrength: Effect strength from 0 to 100%
      • HpvHTEnable: activates/de-activates head tracking usage
      • HpvHTAzimuth: head rotation angle when head tracking is activated in range ]-180,+180]
  • Smart Volume Control incl. automated Dynamic Range Compression, for more details refer to UM1642
      • Target_volume_dB: range -80/+30dB in 1/2dB step
      • Mute: to mute or unmute the signal
      • enable_compr: enable or disable compression
      • attack_time: time it takes the Dynamic Range Compressor to decrease the gain once the signal overshoots a pre-defined level threshold
      • release_time: time it takes the Dynamic Range Compressor to bring the gain back to normal level once the signal falls below a pre-defined level threshold
      • high_quality: the Dynamic Range Compressor level is updated at each input sample (can be updated every 32nd input sample to lower the processor load, at the expense of possible audio quality degradation for some input signals)
  • Gain Manager, for more details refer to UM2006
      • target volume: Digital gain
      • mute: to mute or unmute the signal
  • BAss Manager, for more details refer to UM1778
      • enable: processing of the bass frequencies enable/disable
      • bass_vol: modification of bass signal volume in dB (level from -24 to +24 with 1dB step)
      • freq_xover: modification of bass cut-off frequency (60, 70, 80, 100, 120, 150, 180, 200, 220, 250, 280  and 300Hz)
      • limiter_enable: Limiter enable/disable
      • limiter_release_time: modification of the limiter release time (from 1ms to 1s, default 200ms)
  • Graphical Equalizer, for more details refer to UM1798
      • Enable: Graphical Equalization enable/disable
      • gain_preset_idx: Some Presets are available: User, Pop, Jazz, Rock, Vocal, Classical and HipHop
      • user_gain_per_band_dB: User can change gains of each band from -12db to +12dB for possible number of bands (5, 8 or 10)
  • SDR - Sound Detector, for more details refer to UM2184
    • Enable: processing enable/disable
    • ratio_thr1_dB: threshold (dB) above which the SDR state goes from Inactive to Active in range [+1,+40]
    • ratio_thr2_dB: threshold (dB) below which the SDR state goes from Active to Inactive in range [+1,+40]
    • noise_pwr_min_dB: Minimum noise power below which noise power estimation can't go in range [-87:0]
    • hangover_nb_frame: number of frames of hangover for state going from Active to Inactive.
    • output_state: SDR decision.
  • SMR - Sound Meter, for more details refer to UM2031
    • Enable: processing enable/disable
    • averaging_time: time constant used to smooth the instantaneous level in range [0:10000]ms
    • filter_type: pre-filtering filters [None, A-weighting, C-weighting, DC removal]
    • mean_left_level: sound level of the left channel
    • mean_right_level: sound level on the right channel

  • Embedded Audio Codecs libraries
  • MP3 Decoder (MP3 Dec), for more details refer to Spirit MP3 decoder User Guide
    • Decoder for MPEG-1,2 or 2.5 formats (for layers 1, 2 and 3) supporting constant, free format or VBR variable bit rate for mono or stereo audio input streams with PCM (Pulse Code Modulation) decoded output
  • MP3 Encoder (MP3 Enc), for more details refer to Spirit MP3 encoder User Guide
    • Encoder for MPEG-1,2 or 2.5 formats (for layer 3 only) supporting fixed or free format bit rate for mono or stereo audio input streams

STM32Cube Audio Expansion comes with ready-to-use examples of implementation for the STM32F4xx and STM32F7xx devices. They can be found under \Projects\STM32XX-Discovery\Applications\Audio. These integration examples are provided on STM32746G-Discovery and STM32469I-Discovery boards.

 

The package contains the necessary Drivers (HAL, CMSIS, BSP) and Middleware (Audio, USB) to compile and execute the application.


The HAL drivers provided within this package supports the STM32F44xx/45xx/46xx/47xx and STM32F74xx/75xx/76xx/  77xx lines.
